# ノードの説明

**ノードはワークフローの重要な構成要素です**。異なる機能を持つノードを接続することで、一連のワークフロー操作を実行します。

### コアノード

<Card title="スタート（Start）" href="/ja-jp/guides/workflow/nodes/start">
  ワークフローの開始時に初期パラメータを定義します。
</Card>

<Card title="エンド（End）" href="/ja-jp/guides/workflow/nodes/end">
  ワークフローの最終出力を定義します。
</Card>

<Card title="回答（Answer）" href="/ja-jp/guides/workflow/nodes/answer">
  チャットフローでの応答内容を定義します。
</Card>

<Card title="大規模言語モデル（LLM）" href="/ja-jp/guides/workflow/nodes/llm">
  大規模言語モデルを利用して質問に回答したり、自然言語処理を行います。
</Card>

<Card title="知識検索（Knowledge Retrieval）" href="/ja-jp/guides/workflow/nodes/knowledge-retrieval">
  ナレッジベースからユーザーの質問に関連するテキストを検索し、下流のLLMノードのコンテキストとして使用できます。
</Card>

<Card title="質問分類（Question Classifier）" href="/ja-jp/guides/workflow/nodes/question-classifier">
  分類の説明を定義することで、LLMがユーザー入力に基づいて適切な分類を選択できます。
</Card>

<Card title="条件分岐（IF/ELSE）" href="/ja-jp/guides/workflow/nodes/ifelse">
  if/else条件に基づいてワークフローを2つの分岐に分割することができます。
</Card>

<Card title="コード実行（Code）" href="/ja-jp/guides/workflow/nodes/code">
  Python/NodeJSコードを実行して、ワークフロー内でデータ変換などのカスタムロジックを実行します。
</Card>

<Card title="テンプレート変換（Template）" href="/ja-jp/guides/workflow/nodes/template">
  Jinja2 Pythonテンプレート言語を使用して、柔軟なデータ変換やテキスト処理を行うことができます。
</Card>

<Card title="変数集約（Variable Aggregator）" href="/ja-jp/guides/workflow/nodes/variable-aggregator">
  複数の分岐からの変数を1つの変数に集約し、下流のノードで統一的な設定を可能にします。
</Card>

<Card title="パラメータ抽出（Parameter Extractor）" href="/ja-jp/guides/workflow/nodes/parameter-extractor">
  LLMを使用して自然言語から構造化パラメータを推論・抽出し、後続のツール呼び出しやHTTPリクエストに使用します。
</Card>

<Card title="反復（Iteration）" href="/ja-jp/guides/workflow/nodes/iteration">
  リストオブジェクトに対して複数のステップを実行し、すべての結果を出力します。
</Card>

<Card title="HTTPリクエスト（HTTP Request）" href="/ja-jp/guides/workflow/nodes/http-request">
  HTTPプロトコルを通じてサーバーリクエストを送信し、外部検索結果の取得、webhook、画像生成などのシナリオに適しています。
</Card>

<Card title="ツール（Tools）" href="/ja-jp/guides/workflow/nodes/tools">
  ワークフロー内でDify組み込みツール、カスタムツール、サブワークフローなどを呼び出すことができます。
</Card>

<Card title="変数代入（Variable Assigner）" href="/ja-jp/guides/workflow/nodes/variable-assigner">
  書き込み可能な変数（セッション変数など）に値を代入するために使用します。
</Card>
